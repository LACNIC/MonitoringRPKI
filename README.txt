Los scripts check_RPKIrootOverall, check_RPKIObjectValidation y check_RPKIsyncTime que se encuentran en este repositorio fueron creados para monitorear a través de Icinga el estado del repositorio de RPKI de LACNIC.**check_RPKIrootOverall**El chequeo check_RPKIrootOverall utiliza el servicio Trust Anchor Monitor del Validador de RIPE, recibiendo en la URL el identificador del Trust Anchor del RIR que se desee monitorear.Este chequeo verifica el “Overall Health” del repositorio. Si el “Overall Health” es OK, devuelve un STATE OK. Si el “Overall Health” es ALERT, devuelve un STATE CRITICAL. En cualquier otro caso devuelve un STATE UNKNOWN. Este chequeo no recibe ningún parámetro al ser invocado.**check_RPKIObjectValidation**El chequeo check_RPKIObjectValidation utiliza el servicio Trust Anchor Monitor del Validador de RIPE, recibiendo en la URL el identificador del Trust Anchor del RIR que se desee monitorear.Este chequeo verifica el resultado de la validación de los objetos del repositorio. Si algún objeto no pasó la validación devuelve STATE CRITICAL e informa la cantidad de objetos que no pasaron la validación. Si algún objeto pasó la validación pero con advertencias, devuelve STATE WARNING. Si todos los objetos pasaron la validación y ninguno de ellos pasó la validación con advertencias, devuelve STATE OK. **check_RPKIsyncTime**El chequeo check_RPKIsyncTime hace un rsync recursivo del repositorio de RPKI cuya URL recibe como parámetro y mide el tiempo que demora este rsync.Este chequeo recibe como parámetro la URL de origen para el rsync, la versión (4 o 6) del protocolo IP que se debe utilizar, el umbral de tiempo para declarar estado WARNING, el umbral de tiempo para declarar estado CRITICAL y el RIR al cual corresponde el repositorio que se está midiendo.Para evitar que los chequeos colisionen usando todos como destino el mismo directorio en el directorio /tmp, el nombre de los directorios de destino se define al momento de realizar el chequeo utilizando un timestamp como parte del nombre.El comando rsync recibe como parámetro “r” para que sea recursivo y 4 o 6, dependiendo de lo que se haya recibido como parámetro del chequeo.El tiempo que demora el rsync se mide utilizando el comando “time”. Al finalizar el rsync se elimina el directorio de destino.Si el tiempo que demoró el rsync es menor al umbral de WARNING, el chequeo devuelve STATE OK. Si el tiempo está entre el umbral de WARNING y el umbral de CRITICAL, el chequeo devuelve STATE WARNING. Si el tiempo es mayor al umbral de CRITICAL, el chequeo devuelve STATE CRITICAL. En cualquier otro caso el chequeo devuelve STATE UNKNOWN.
